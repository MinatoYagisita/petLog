# テスト実施・証跡整理 Skill

> 前提: Vitest（単体・結合）/ Playwright（E2E・受入）/ pnpm

## この Skill を使う場面

- テスト仕様書に基づいてテストを実施するとき
- テスト結果の証跡を整理するとき
- テスト結果サマリを PM に報告するとき

## 入力

- テストケース一覧（test-spec-creator の出力など）
- テスト対象環境の情報
- テスト実施者名

## 実施手順

### Step 1: テスト準備

1. テストケース一覧を確認する
2. テスト環境が正しくセットアップされているか確認する
   - `pnpm install` が完了しているか
   - テスト用 DB が準備されているか（`prisma migrate dev`）
   - `.env.test` 等のテスト用環境変数が設定されているか
3. テストに必要なデータを準備する（`prisma/seed.ts` またはテスト内セットアップ）

### Step 2: テスト実施

1. Vitest 対象のテストを実行する（`pnpm test`）
2. Playwright 対象のテストを実行する（`pnpm test:e2e`）
3. 各ケースの結果を記録する（OK / NG / 保留）
4. NG の場合は不具合記録を作成する
5. 証跡を保存する（Vitest のログ、Playwright のスクリーンショット・トレース等）

### Step 3: 結果の整理

1. テスト結果一覧を更新する
2. 不具合一覧を整理する
3. テスト結果サマリを作成する

## 出力フォーマット

### テスト結果一覧

```markdown
| TC-ID | テスト項目 | 期待結果 | 実施結果 | 実施日 | 実施者 | 備考 |
|-------|-----------|---------|---------|--------|--------|------|
| TC-001 | 有効な情報で登録できる | 登録成功 | OK | 2024-01-15 | 田中 | |
| TC-002 | 必須項目未入力 | エラー表示 | NG | 2024-01-15 | 田中 | 不具合 BUG-001 |
| TC-003 | 文字数上限ちょうど | 登録成功 | OK | 2024-01-15 | 田中 | |
```

### 不具合記録テンプレート

```markdown
## BUG-001: [不具合タイトル]

- **対象テストケース**: TC-002
- **重要度**: 高 / 中 / 低
- **対象機能**: ユーザー登録画面
- **発生環境**: ステージング / Chrome 120
- **再現手順**:
  1. /register にアクセスする
  2. メールアドレスを空欄のまま登録ボタンを押す
- **期待結果**: 「メールアドレスは必須です」のエラーが表示される
- **実際の結果**: 500エラーが発生する
- **再現率**: 毎回
- **スクリーンショット**: [添付 or パス]
- **発見日**: 2024-01-15
- **発見者**: 田中
- **ステータス**: 未対応 / 対応中 / 修正済み / 再テスト済み
```

### テスト結果サマリ

```markdown
- **対象機能**: ユーザー登録機能
- **テスト期間**: 2024-01-15 〜 2024-01-16
- **テスト実施者**: 田中
- **テスト環境**: ステージング環境

### 結果概要

| 項目 | 件数 |
|------|------|
| 総テストケース数 | 20 |
| OK | 17 |
| NG | 2 |
| 保留 | 1 |

### 不具合一覧

| BUG-ID | 重要度 | 概要 | ステータス |
|--------|--------|------|-----------|
| BUG-001 | 高 | 必須チェック未実装 | 未対応 |
| BUG-002 | 中 | エラーメッセージが英語 | 未対応 |

- TC-015: 外部API連携のテストは本番相当環境でのみ実施可能

```

## 再試験ルール

- 不具合修正後は、該当テストケースを再実施する
- 再テスト結果は元の結果を上書きせず、再テスト結果として追記する
- 修正による影響範囲が広い場合は、関連テストケースも再実施する
- 再テスト結果もサマリに含める

## チェック観点

- [ ] すべてのテストケースを実施したか
- [ ] NG のケースには不具合記録を作成したか
- [ ] 再現手順は第三者が再現できる粒度か
- [ ] テスト結果サマリを作成したか
- [ ] 保留事項を明記したか

## 禁止事項

- テストケースに無いテストを実施して「OK」にしない（観点追加は別途相談）
- NG を「軽微だから」と記録せずにスキップしない
- 不具合の再現手順を省略しない
- テスト結果を事後にまとめて記録しない（実施時にリアルタイムで記録する）
