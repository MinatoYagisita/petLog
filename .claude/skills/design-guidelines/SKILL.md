# デザインガイドライン Skill

> 前提: React + Tailwind CSS / Next.js
> UIルール詳細は `.claude/rules/ui.md` を参照。

## この Skill を使う場面

- 画面の UI/UX レビューを行うとき
- 新規画面の設計時に品質基準を確認するとき
- お客様から「使いにくい」という指摘を受けたとき

## 入力

- レビュー対象の画面（スクリーンショット、URL、またはコード）
- 画面の用途・業務フロー
- 対象ユーザーの情報（あれば）

## 実施手順

### Step 1: 一覧画面のチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | データ件数表示 | 全件数が画面に表示されているか |
| 2 | ページネーション | 大量データ時にページ分割されるか |
| 3 | ソート機能 | 主要カラムでソート可能か |
| 4 | 0件表示 | データなしのとき「該当なし」メッセージが出るか |
| 5 | ローディング | データ読込中にスピナー等が表示されるか |
| 6 | 行の識別 | 各行が何のデータか一目で分かるか |
| 7 | アクション導線 | 詳細・編集・削除への導線が分かりやすいか |
| 8 | 固定列 | 横スクロール時にID列等の固定が必要なケースを考慮しているか |
| 9 | 列の可読性 | 主要列が折り返しや省略で読めなくなっていないか |
| 10 | 情報密度 | 1画面内の情報量が多すぎないか |
| 11 | 行操作の明確さ | 行クリックとボタン操作の役割が曖昧でないか |

### Step 2: 検索 UI のチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | 条件配置 | よく使う条件が上部に配置されているか |
| 2 | 検索・クリア | 検索ボタンとクリアボタンがあるか |
| 3 | 0件結果 | 検索結果0件のときにメッセージが出るか |
| 4 | 条件保持 | 画面遷移後に検索条件が保持されるか |
| 5 | 入力補助 | 日付ピッカー、セレクトボックス等が適切か |
| 6 | 最小操作 | よく使う条件が最小操作で入力できるか |
| 7 | グルーピング | 検索条件が多い場合、グルーピングや折りたたみが必要か |
| 8 | 初期表示 | 初期表示時の検索条件が妥当か |
| 9 | 件数表示 | 絞り込み結果の件数や状態が分かるか |

### Step 3: 入力フォームのチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | 必須表示 | 必須項目に `*` や「必須」ラベルがあるか |
| 2 | 入力例 | プレースホルダーや入力例が設定されているか |
| 3 | エラー位置 | エラーメッセージが該当項目の近くに表示されるか |
| 4 | 離脱防止 | 入力途中の離脱に確認ダイアログが出るか |
| 5 | 成功通知 | 保存成功時にフィードバックが表示されるか |
| 6 | 二重送信防止 | 送信中にボタンが無効化されるか |
| 7 | ラベル位置 | ラベル位置（上 / 左 / 内）が統一されているか |
| 8 | 必須・任意の区別 | 必須と任意の区別が一目で分かるか |
| 9 | セクション分け | フォームが長い場合、区切りやセクション分けがあるか |
| 10 | 入力順 | 入力順が自然か（業務フローに沿っているか） |
| 11 | 補助説明 | 入力ミスしやすい項目に補助説明があるか |

### Step 4: ボタン配置のチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | 主要アクション位置 | 保存・登録ボタンが右下 or 右上にあるか |
| 2 | 破壊的アクション | 削除ボタンが主要アクションから離れているか |
| 3 | 削除確認 | 削除操作に確認ダイアログがあるか |
| 4 | ラベル | 「OK」ではなく「保存する」など具体的か |
| 5 | 役割の区別 | Primary / Secondary / Danger の役割が見た目で区別できるか |
| 6 | 配置ルール統一 | 戻る・キャンセル・保存の配置ルールが画面間で統一されているか |
| 7 | 二重押下防止 | 処理中のボタン無効化・ローディング表示が考慮されているか |

### Step 5: エラー表示のチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | ユーザー向け | 技術的なエラーコードだけでなく対処法が示されるか |
| 2 | 視覚的区別 | 致命的エラーと入力エラーが見た目でも区別できるか |
| 3 | データ保持 | エラー発生時に入力済みデータが消えないか |
| 4 | 次の行動 | エラーメッセージが次の行動につながる内容か |
| 5 | 複数エラー | 複数項目エラー時に、どこを直せばいいか分かるか |

### Step 6: 余白・タイポグラフィのチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | 余白の一貫性 | 要素間の余白が統一されたルールに従っているか |
| 2 | グルーピング | 関連要素がグループ化されているか |
| 3 | フォントサイズ | 本文が最低14pxあるか |
| 4 | 行間 | 1.5〜1.75程度の行間があるか |
| 5 | 横幅 | テキストが広がりすぎていないか（max-width） |
| 6 | 階層の視認性 | 見出し・本文・補足の階層が視覚的に分かれているか |
| 7 | テーブル内の詰まり | テーブルやフォーム内で文字が詰まりすぎていないか |
| 8 | 長文表示 | 画面幅に対して読みにくい長文表示になっていないか |

### Step 7: 一貫性のチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | ボタンラベル | 同じ意味のボタンが同じラベルか |
| 2 | 日付フォーマット | 画面間で統一されているか |
| 3 | ステータス表示 | ラベル・色が画面間で統一されているか。ステータスの意味が画面ごとに変わっていないか |
| 4 | 用語 | 同じ概念に同じ用語を使っているか |
| 5 | アイコン・文言・色 | 同じ操作に同じアイコン・文言・色を使っているか |
| 6 | モーダル・トースト・ダイアログ | これらの扱いが画面間で統一されているか |

### Step 8: アクセシビリティのチェック

| # | 確認項目 | 基準 |
|---|---------|------|
| 1 | alt テキスト | 画像に alt が設定されているか |
| 2 | label 紐付け | フォーム要素に label が紐付いているか |
| 3 | キーボード操作 | Tab キーでフォーカス移動でき、フォーカスが見えるか |
| 4 | フォーカス順 | フォーカス順が自然か（DOM順と視覚的な順序が一致しているか） |
| 5 | モーダルのフォーカス | モーダル表示時にフォーカスが閉じ込められているか |
| 6 | コントラスト | WCAG AA 基準（4.5:1以上）を満たしているか |
| 7 | 色以外の手がかり | 色だけで情報を伝えていないか |
| 8 | スクリーンリーダー対応 | エラーがスクリーンリーダーでも把握できる形（aria-live等）になっているか |
| 9 | プレースホルダー依存 | プレースホルダーだけで意味を伝えていないか（入力後に消えて分からなくならないか） |

## 出力フォーマット

```markdown
## UI/UX レビュー結果

- **対象画面**: ユーザー一覧画面
- **レビュー日**: 2024-01-20
- **レビュアー**: [名前]

### チェック結果

| カテゴリ | 確認数 | OK | NG | 該当なし |
|----------|-------|----|----|---------|
| 一覧画面 | 7 | 6 | 1 | 0 |
| 検索UI | 5 | 4 | 1 | 0 |
| 入力フォーム | 6 | - | - | 6 |
| ボタン配置 | 4 | 4 | 0 | 0 |
| エラー表示 | 3 | 2 | 1 | 0 |
| 余白・タイポ | 5 | 5 | 0 | 0 |
| 一貫性 | 4 | 3 | 1 | 0 |
| アクセシビリティ | 5 | 4 | 1 | 0 |

### 指摘一覧

| # | カテゴリ | 該当箇所 | 指摘内容 | 重要度 | 影響 | 改善提案 |
|---|---------|---------|---------|--------|------|---------|
| 1 | 一覧画面 | ユーザー一覧 全体 | 0件時にメッセージが表示されない | 中 | 操作性 | 「該当するデータがありません」を表示 |
| 2 | 検索UI | ユーザー検索 条件エリア | 検索条件が画面遷移で消える | 高 | 操作性 | URL パラメータ or state で保持 |
| 3 | エラー表示 | 全画面 エラーハンドリング | 500エラー時に技術的メッセージが表示される | 高 | 誤操作防止 | ユーザー向けメッセージに差し替え |
| 4 | 一貫性 | 一覧と詳細画面の日付表示 | 日付が YYYY/MM/DD と YYYY-MM-DD 混在 | 低 | 一貫性 | YYYY/MM/DD に統一 |
| 5 | アクセシビリティ | ユーザー検索 検索ボタン | 検索ボタンのコントラスト不足 | 中 | アクセシビリティ | ボタン色を調整 |

> 影響欄は「操作性 / 誤操作防止 / 一貫性 / アクセシビリティ」のいずれかを記入する。
```

## チェック観点

- [ ] 全8ステップのチェックを実施したか
- [ ] 「見た目」ではなく「業務で使いやすいか」の視点で確認したか
- [ ] 指摘に具体的な改善提案を含めたか
- [ ] アクセシビリティを確認したか
- [ ] 画面間の一貫性を確認したか

## 禁止事項

- デザインの好みで指摘しない。**レビュー基準は美的好みではなく、業務効率・誤操作防止・理解しやすさ・一貫性**である
- 見た目だけ整えて操作性を無視しない
- アクセシビリティを「後回し」にしない
- フレームワーク標準のコンポーネントがあるのに独自実装しない。ただし、**標準のままでは業務に不向きな場合は調整点を指摘する**
- PM やデザイナーの判断が必要な事項を勝手に決定しない
